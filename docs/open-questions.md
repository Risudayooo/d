# 未確定事項の技術検討(11章)

各項目について「技術観点での推奨案」と根拠を示す。**確定ではなく提案**。最終決定はりすさん判断。

---

## ✅ ディスクの初期種類数
**確定済み**: House / Dubstep / DnB の3種で仮置き。`DiscType` enum + `DiscAbility` 実装で骨格化済み(Phase 1 スコープ)。

---

## 1. 銃本体(クラス)の種類分け

**推奨: 開始は3クラス。** ディスク3種と直交させ、「クラス × ディスク」で 3×3 の手触りを早期に検証する。

| クラス | 役割 | 射撃の根幹 | 想定 |
|---|---|---|---|
| **Pulse**(基準) | 遠隔・万能 | 中距離ヒットスキャン | クセがなくチュートリアル向き |
| **Breaker**(近接寄り) | 高 DPS・短射程 | ショットガン的拡散 / 接近戦 | リスク・リターン高め |
| **Drone**(特殊型) | 設置 / 持続 | 弾が貫通・残留、または自動ターレット的挙動 | ビルド構築型 |

- 根拠: Risk of Rain のサバイバー数の初期目安(3〜4)に合わせ、ディスクとの組み合わせ爆発を抑えつつ多様性を確保。
- 実装: ディスクと同じく `GunClass` を `GunCDPlayerItem` のデータコンポーネントに保持。発射ロジックは `class の基礎挙動 → disc の修飾` の二段で合成(Strategy パターン)。

---

## 2. ダンジョンクロールの部屋数 / 構造

**推奨: 固定パターンの部屋プレハブ + グラフ接続のハイブリッド。**

- **部屋自体**: 手作業の固定プレハブ(構造ブロック / NBT structure)を **種別ごとに 5〜8 個** 用意。完全手続き生成はコスト過大。
- **接続**: フロアを有向グラフで生成。1フロア **6〜10 部屋**、深さ 3〜4。Slay the Spire 型の分岐マップが意思決定密度を出しやすい。
- **フロア数**: 1ラン **3〜5 フロア**(朝→夜の演出と同期)。各フロア末尾にボス部屋。
- 実装段階: Phase 1 はまず「固定の小さなテスト ダンジョン(戦闘→資源→ボスの一本道)」で戦闘ループを検証 → Phase 2 でグラフ生成に拡張。

---

## 3. 交換スポット(ジュークボックス的 UI)の操作フロー

**推奨フロー**:
1. 資源部屋にジュークボックス型ブロック(`disc_exchange_station`)を設置。
2. プレイヤーが近接して **使用キー** で起動 → 専用 Screen(GUI)を開く。
3. GUI に「所持ディスク一覧」+「現在装填中スロット」を表示。ドラッグ or クリックで装填ディスクを差し替え。
4. 確定時に銃のデータコンポーネントを更新し、BGM を新ディスクのものへクロスフェード。
5. 戦闘中の即時切替は不可(ステーション専有)で、設計意図(選曲 = 意思決定の場)を担保。

- 実装: `NamedScreenHandlerFactory` + `ScreenHandler` + クライアント `HandledScreen`。Phase 1 は最小限(リスト + 1スロット差し替え)。

---

## 4. ブリンクの距離・クールダウン

**推奨初期値(要プレイテスト)**:
- 距離: **6 ブロック**(壁貫通させず、最初の衝突ブロック手前で停止 = レイキャスト)。
- クールダウン: **2.5 秒(50 tick)**。連続スピード感とリスク管理の両立。
- 上方向制限: 視線方向そのままだと飛びすぎるので、Y 成分に 0.6 倍の減衰をかける案。
- 無敵フレーム: ブリンク中 3 tick の i-frame を付けると回避技として機能(Phase 2)。
- 値は `gradle.properties` ではなくゲーム内 config(`discgun.json`)で調整可能にしておくと反復が速い。

---

## 5. パーク / スキル / 消費アイテム / 武器MOD の内容

**推奨: Phase 2。** iceLatte を参考に各レイヤー初期 4〜6 個。たたき台:

- **パーク(常時)**: 移動速度 +10% / クリティカル率 +8% / 拾得クレジット +25% / リロード速度 +15% / 二段ジャンプ→三段化。
- **スキル(CD あり)**: 緊急回復、範囲スロウ、自己バフ(攻撃速度)、おとり設置。
- **消費アイテム**: フラググレネード、即時回復、一時シールド、移動速度ポーション。
- **武器MOD**: 炎上付与、感電(連鎖)、ライフスティール 5%、貫通弾、弱点ダメージ +30%。
- 実装: 全レイヤーを共通の `RunModifier` インターフェース(ステータス集計 + イベントフック)に集約し、ビルド計算を一元化。

---

## 6. チャレンジ部屋の条件と報酬

**推奨: Phase 2。** リスク・リターンを明示的に提示する一回性イベント。たたき台:

| 条件 | 報酬目安 |
|---|---|
| 被ダメージ +50%(部屋内) | 高レアリティ ディスク確定 |
| 時間制限 60 秒で全敵撃破 | クレジット 2倍 + 武器MOD |
| ノーダメージ クリア | パーク選択(3択) |

- 入室前に条件と報酬をプレビューし、プレイヤーに「受けるか」を選ばせる(ドアでの選択 UI)。

---

## 決定が必要な順序(おすすめ)

1. **ブリンク数値(#4)** — Phase 1 の手触りに直結、すぐ確定したい。
2. **テストダンジョン構造(#2 の Phase 1 部分)** — 戦闘ループ検証に必要。
3. **銃クラス数(#1)** — Phase 1 後半。まず Pulse 1 種で射撃を固め、その後拡張。
4. 交換 UI(#3)・強化レイヤー(#5)・チャレンジ(#6)は Phase 2。
