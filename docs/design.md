# 企画仕様書 — タイトル未定(working title: **DiscGun**)

> 本書は提供されたドラフトを整形した正式版。`docs/open-questions.md` に未確定事項の技術検討をまとめる。

## 1. コンセプト

銃型CDプレイヤーに好きな曲(ディスク)を差し込み、その曲の個性をそのまま戦闘能力として戦う、ローグライク・スピードシューター。

- **プラットフォーム**: Minecraft MOD(**Fabric** / MC 1.21.1 / Java 21)
- **配布形態**: `.mrpack`(Modpack)形式。FancyMenu で専用タイトル画面 / ワールド選択 UI に差し替え、「起動したら専用ゲームとして直接プレイできる」体験を目指す
- **差別化軸**: リズム判定でアクションが強化される(ROBOBEAT 型)のではなく、**ディスクを選ぶ / 差し替える行為そのものがコアループ**。BGM と能力が一体化した「選曲 = ビルド」のローグライク
- **参考**: 別企画「Project:iceLatte」(Rogue Lite / Hack and Slash / FPS Dungeon Crawler)の設計思想を一部統合。「マイクラ離れ」のビジュアル / アクションと、ランごとの意思決定密度・爆発的成長の実感を継承

## 2. プレイヤーアクション(スピードシューター要素)

| アクション | 実装方針 | 難易度 | Phase |
|---|---|---|---|
| ダッシュ(ブリンク) | 短距離テレポート / 瞬間移動(Speed 効果ではなく位置移動寄り)。クールダウン管理(コンポーネント / スコアボード) | 低〜中 | 1 |
| スライディング | しゃがみ中に前進速度ブースト + 当たり判定高さ変更 | 低〜中 | 1 |
| 二段ジャンプ | 空中ジャンプ回数カウント + Y 方向速度付与 | 低〜中 | 1 |
| リコイルジャンプ | 射撃イベント内で下方向ショット時に上方向速度付与 | 中 | 1〜2 |
| ウォールラン | 壁検出(レイキャスト / 周囲ブロック判定)+ 移動ベクトル計算 | 高 | 2 |

ダッシュ / スライディング / 二段ジャンプは「プレイヤーの速度ベクトルを直接操作する」同一の技術パターンの応用。1つ確立すれば横展開しやすい。

## 3. 銃型CDプレイヤー(武器システム)

- **見た目**: 銃型のCDプレイヤー。ダーク・ストリート・Y2K グランジ的質感(ボロボロのボムボックス的デザイン)
- ディスクは武器側のスロットに差し込む(プレイヤー本体の改造ではなく、アイテムのデータコンポーネントで管理)
- **一人称の腕表現**: TACZ 同様 GeckoLib を使用。腕は別グループとしてモデリングし、ピボットをモデル底面中心に配置、回転はアニメーション側で制御。**Phase 2** の拡張要素

### 進行構造(二層構造) — Risk of Rain 的

- **銃本体(クラス)**: ラン開始時に1つ選択し、**ラン中は変更不可**。プレイスタイルの根幹(近接寄り / 遠隔寄り / 特殊型など)を決定
- **ディスク**: ラン中、資源 / ショップ部屋や専用スポットで入手・交換可能。曲の個性 = 戦闘能力として、銃本体の上に乗る可変要素

「何を選んだか」が方向性を決め(銃)、「ラン中どう転がるか」が個々のランの個性を作る(ディスク)。

## 4. ディスクシステム

初期実装は **House / Dubstep / DnB の3種** を仮置きとしてプロトタイプ。本格的なジャンル選定は後回し。

各ディスクが保持するもの:
- 固有 BGM(差し込み時に再生開始、Java 側でサウンドイベント制御)
- **通常攻撃(左クリック)のタイプ**
- 右クリック固有アクション
- 基本性能の傾向(連射 / 威力 / 支援などのキャラ付け)

**実装方針**: 「能力インターフェース」(`DiscAbility`)を1つ定義し、各ディスクがそれを実装(左クリック挙動・右クリック挙動・BGM を1つのディスクが保持)。最初は3種で骨格を確認してから増やす。

### ディスク別 通常攻撃(左クリック)

| 系統 | 通常攻撃タイプ | パラメータ目安 |
|---|---|---|
| House | 一定テンポの安定した連射(基準点) | 中威力 / CD 5t |
| Dubstep | ため→ドロップ式。チャージ後に高威力の一撃 / 範囲 | 高威力 / CD 18t |
| DnB | 高速連射・低威力(刻みの細かいリズム) | 低威力 / CD 2t |

### ディスク別 右クリックアクション

| 系統 | 右クリックアクション |
|---|---|
| House | 短距離テレポート(軽快な移動) |
| Dubstep | 一時的ダメージシールド展開(ドロップ前の溜め) |
| DnB | 周囲を一瞬かき乱す範囲攻撃(ブレイクビーツの瞬発力) |

### ディスクの役割(育成・所有の快楽)

- ラン中にドロップで入手 → 所持ディスクの組み合わせがそのランのビルド
- CD コレクター的な「所持すること自体の満足感」をゲームループに組み込む

## 5. 操作系統(キーバインド)

標準 FPS の操作感をベースに、Minecraft 標準キーとの衝突を **銃を持っている間だけ** オーバーライド。

| 操作 | キー | 備考 |
|---|---|---|
| 移動 | WASD | 標準 |
| ジャンプ / 二段ジャンプ | Space | 標準 + 拡張 |
| しゃがみ / スライディング | Shift | 標準 + 拡張 |
| 射撃 | 左クリック | 銃所持中、Attack をオーバーライド |
| ディスク固有アクション | 右クリック | ディスクごとに分岐 |
| パリィ | F | バニラのオフハンド切替を無効化し専有 |
| リロード | R | 新規 KeyMapping |
| ブリンク | ダッシュ / スプリントキー | 瞬間移動的な短距離ブリンクとして再定義 |

※ ディスク選択(モード切替)は戦闘中いつでもは不可。マップ上の特定スポットでのみ選択可能(6章)。

## 6. ローグライク構造

- ランダム性 / 育成要素は維持。パーマデス(死亡時ランリセット、メタ進行は薄め)
- プロシージャル生成: 部屋の繋ぎ方をランダム化(部屋自体は固定パターンの組み合わせから)
- ディスクの組み合わせ自体がビルド
- **ディスク交換**: マップ上の専用スポット(ジュークボックス的交換ポイント)でのみ可能(A 確定)
- ステージ構成: **ダンジョンクロール型**

### 部屋種別

| タイプ | 内容 | Phase |
|---|---|---|
| 戦闘 | 敵と戦う(通貨・回復・チャレンジ条件付きバリエーションも) | 1 |
| 資源 | アイテム / ディスク / パーク入手。ディスク交換スポットを配置 | 1 |
| ショップ | クレジットとアイテムを交換 | 2 |
| イベント | 特殊な一回性の出来事 | 2 |
| チャレンジ | 追加条件と引き換えに高額報酬 | 2 |
| ボス | 討伐して次フロアへ | 1 |

### ラン中の強化レイヤー(三層構造)

**銃(固定)+ ディスク(可変・曲の個性)+ パーク / スキル / アイテム / 武器MOD(汎用強化)**

- **パーク**: ラン中ずっと効く自己強化(移動速度、クリティカル増加、報酬増加 等)
- **スキル**: クールダウンありの補助アクション(ダメージ / バフ / 回復)
- **消費アイテム**: 使い切り(グレネード、即時回復、バフ)
- **武器MOD**: 銃本体への付加効果(属性付与、ダメージボーナス、ライフスティール)

## 7. 世界観・演出

- ベース美学: vaporwave / webpunk、都会的な舞台。ダーク・ストリート・Y2K グランジと接続
- 時間帯と難易度の同期: 朝→夜へフロア進行で難易度上昇
- フィードバック: クリティカル音 / ダメージ数値、アメコミ風オノマトペポップアップ、フラッシュ / モーションブラー(Phase 2)
- クリティカル(弱点): FPS 視点を活かし頭部などの弱点ヒットで明確な演出

## 8. パリィシステム

- **F キーで発動**(確定。左右持ち替え式は不採用)。発動後数 tick(0.2〜0.3 秒)だけ「パリィ判定中」フラグ
- 簡略化: バニラのオフハンド切替機能自体を無効化し、F キーを常にパリィ専用に固定
- 判定中に攻撃(プロジェクタイル / 近接)を受けたらダメージ無効化 + スタン / 反射等
- Phase 1: シンプル版(無効化のみ)/ Phase 2: 攻撃種別ごとの反応分岐

## 9. 配布・体験設計

- Java MOD(Fabric)+ データパック / リソースパック + FancyMenu を `.mrpack` で一括パッケージ化
- 起動直後から専用ゲームとして遊べる体験(Atmos 参考)

## 10. Phase 分け

### Phase 1(MVP) — 本リポジトリの初期スコープ
- 基本射撃 + House / Dubstep / DnB の3ディスク(BGM 切替・右クリックアクション含む)
- ブリンク / スライディング / 二段ジャンプ
- パリィ(シンプル版・F キー)
- 銃本体固定 + ディスク交換(専用スポットのみ)の二層構造
- 部屋種別: 戦闘 / 資源 / ボスの最小構成
- 基本ヒットフィードバック(ダメージ数値、ヒット音)
- 一人称表現は通常アイテムモデルでよい(GeckoLib 腕は見送り)

### Phase 2(拡張)
- ディスク種類追加 / GeckoLib 一人称腕 / スクラッチ操作 / ウォールラン
- パリィの攻撃種別別リアクション / パーク・スキル・消費アイテム・武器MOD レイヤー
- 部屋種別拡充 / クリティカル(弱点)システム / 演出強化 / 時間帯難易度同期
